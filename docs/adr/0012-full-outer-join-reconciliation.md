# Full outer join for Bank/Partner reconciliation, not inner join

The Transaction fact is built by full-outer-joining Bank-side and Partner-side records on the shared gateway-assigned Transaction ID (ADR-0009), so a transaction appears in the fact table as soon as either side has reported it — with null attributes on whichever side hasn't arrived yet — rather than waiting for both sides to match before the row is visible at all. This was chosen because the two sides genuinely arrive on different schedules as normal operation, not just as an anomaly; an inner join would understate same-day volume until the slower side catches up, which is misleading for an operational/performance view. The trade-off: dashboard/dbt queries that compute settlement rate or other Bank-side-dependent metrics must explicitly handle partial rows (null Bank-side data) rather than assuming every visible row is complete.
