# Denormalize the applied Fee Schedule rate onto the Transaction fact

The fee amount (and the specific rate that produced it) is calculated once when a transaction is captured and stored directly on the Transaction fact row, rather than computed at query time by joining transactions to a "current rates" Fee Schedule table. This was chosen deliberately over a Type 2 SCD Fee Schedule dimension with effective-dating, because a naive join against "current" rates would silently rewrite historical revenue whenever a rate changes — and a full SCD dimension would reintroduce the history-tracking machinery that ADR-0005 deliberately avoided everywhere else in the warehouse. The Transaction fact stays self-contained and historically accurate by construction: a future reader who's tempted to "simplify" this by joining to a live rates table should not do so.
