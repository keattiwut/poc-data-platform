# Never store raw PAN/account numbers — masked/tokenized only

Bank and Partner source systems are required to send only masked/tokenized card and account identifiers (e.g. last-4 digits, a token, or a hashed reference) — never full PANs or raw bank account numbers — and this is treated as a hard integration requirement, not a pipeline-side responsibility to mask after the fact. This was chosen deliberately over masking/tokenizing on ingestion (which would put PCI-DSS-relevant logic inside the pipeline itself) because it keeps the entire data platform out of PCI-DSS scope — the pipeline never touches true cardholder data at any stage, from Airbyte extraction through the lake, warehouse, and Superset dashboards. This must be validated for every future real Bank/Partner integration before it's connected; "we'll add masking later" is a far more expensive retrofit than requiring it upfront.
